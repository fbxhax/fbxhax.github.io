Fruity Dream Booth

The Fruity Dream Booth initial concept was to send pictures taken by a Raspberry Pi based photo-booth through the Google DeepDream library and tweet out the results.

The challenges and computing requirements for Google Deep Dream made doing this unfeasible on a Raspberry Pi. hacking project learning through pain success

The project team for Fruity Dream Booth:

  • ferrio - core web server
  • teknofire - javascript tweaking and moral support
  • vlraymond - twitter data feed setup, coding and passion
  • dayne - general idea, infrastructure, and glue

The project code is hosted at dayne/fruitydreambooth
